# CourseList 项目功能说明

## 项目定位

CourseList 是一个基于 WinUI 3 的课程管理与课表展示应用，核心目标是帮助用户快速录入课程、可视化查看课程安排，并提供首页今日课程概览与个性化设置能力。

## 技术栈

- **运行时**：.NET 8（`net8.0-windows10.0.19041.0`），目标平台最低 **Windows 10 版本 17763**。
- **UI**：WinUI 3、Windows App SDK（项目引用约 1.8 分支）。
- **其他**：CommunityToolkit.WinUI.Notifications（Toast）、WebView2（教务导入）。

## 代码与目录结构（概要）

- **`Models/`**：领域模型 `Course.cs`（课程与 UI 辅助属性）、`WeekScheduleOverride.cs`（按周课表覆盖）。
- **`Views/`**：各 WinUI 页面与教务导入窗口，详见 `Views/README.md`。
- **`Helpers/`**：业务与基础设施逻辑，按职责分子目录，**命名空间仍为 `CourseList.Helpers`**，详见 `Helpers/README.md`。
  - `Helpers/Core/`：数据根路径、多方案、`config.json` / `courses.json`
  - `Helpers/Schedule/`：按周有效课表、周覆盖文件、学期周次、冲突检测
  - `Helpers/Import/`：教务 WebView 会话与课表 JSON 解析
  - `Helpers/Ui/`：主题、对话框守卫、关闭行为弹窗、窗口模式控制、系统通知
  - `Helpers/Platform/`：系统托盘、窗口消息与 DPI/最小尺寸互操作

## 已实现功能

### 0. 近期修复与更新（2026-04）

- **课表页面（SchedulePage）**
  - 桌面大课表在窗口拉伸时增加节流重绘与阈值判断，显著降低卡顿与重复渲染。
  - 小窗口（compact）模式下提升行高与课程块高度，改善可读性。

- **待办页面（ToDoPage）**
  - 大窗口与小窗口采用不同卡片布局：桌面保持网格卡片，compact 下切换为横向单列条目卡片。
  - 待办卡片右上角操作区改为相对定位（Pin + 紧急度），不依赖最小宽度硬撑。
  - 卡片内容区新增自适应纵向滚动，内容/标签较多时可向下滚动查看，避免标题与正文被挤压。
  - 待办卡片布局参数对齐课程列表风格：支持随容器宽度自适应填充与换列。

- **置顶待办窗口（TodoPinWindow）**
  - 置顶窗口尺寸口径统一为窗口尺寸（DIP）读写，修复“保存值与实际显示值不一致（尤其高度偏差）”问题。
  - 移除最小宽高强制限制，仅保留非法值兜底与上限约束。
  - 新增“尺寸调整模式”叠层，支持在设置页打开专用调整窗口并回写配置。
  - 尺寸调整叠层文案布局改为 Grid，并移除顶部标题文案，保留说明、实时尺寸与确认按钮。

- **设置页面（SettingsPage）**
  - 布局扩展为三列：系统设置 / 课程表功能 / 置顶待办窗口。
  - 三列改为 `Auto` 宽度并保留右侧弹性空白列，窗口变宽时列位置稳定不漂移。
  - 桌面模式下第二、三列设置统一左间距；compact 模式下自动清零边距，确保与第一列左对齐。
  - 新增“调整置顶待办窗口大小”入口：打开可拉伸预览窗口，确认后写入全局配置。

### 1. 首页（HomePage）

- 显示欢迎信息、当天星期、今日课程概览。
- 显示“今日课程”卡片列表（桌面宽度下与左侧概览**左右分栏**；**紧凑模式**下改为**上下堆叠**，适配窄窗与竖屏）。
- 卡片采用左侧色条 + 右侧信息样式，直观区分课程颜色。
- 当天课程会按节次自动排序。
- 对于同一课程在同一天的**不连续节次**，会拆分为多条课程安排分别显示。
- 时间展示支持紧凑格式，如 `08:00 ~ 09:40`。
- 切换课表方案后，首页数据随当前方案刷新。

### 2. 课程列表（CourseListPage）

- 课程增删改查（新增、编辑、删除）。
- 卡片式课程展示（固定卡片高度，卡片**内部**可滚动查看长文本），支持显示：
  - 课程名
  - 教师
  - 教室
  - 上课时间
  - 备注
- 列表区域外包 **`ScrollViewer`**：卡片较多超出窗口高度时出现**纵向滚动条**，避免内容被裁切且无滚动。
- 支持搜索（名称/教师/教室/备注/时间文本）。
- 支持筛选：
  - 星期筛选
  - 周类型筛选（全周、单周、双周）
  - 节次筛选
- 卡片选中高亮，便于编辑/删除操作。

### 3. 课程表页（SchedulePage）

- 网格式课程表展示（支持周一到周五 / 周一到周日两种模式，由设置决定）。
- 根据“每日节数”动态生成行数。
- 首列显示“节次 + 对应时间”。
- 浅色/深色下表头与背景使用页面内主题色，与系统主题协调。
- 课程渲染支持连续节次合并：
  - 连续节次视觉合并为一个课程块
  - 中间缝隙不再断开
  - 课程名称/教师/教室只显示一次
- 课程选中与交互：
  - 点击课程可选中并高亮
  - 双击课程可直接进入编辑
  - 点击空白区域取消选中
- 底部周次切换（上一周/下一周/回到本周），并显示是否“非本周”提示。
- 悬浮操作栏支持展开/收起动画（新增、编辑、删除）。
- **拖放排课**：将课程块拖到其他节次/星期；松手后可选「仅当前课表页显示的第 N 周」或「全局」生效，并进行时间冲突检测；拖拽过程含预览与平滑过渡。
- **按周调课**：仅本周的排课保存在当前方案目录的 `week-overrides.json`，不修改全局课程节次；全局调课则写入 `courses.json` 并移除该周对应覆盖。

### 4. 课程录入与编辑（CourseFormPage）

- 支持录入课程基础字段：课程名、教师、教室、星期、节次、颜色、周类型、**学期内起止周次**（`FromWeek` / `ToWeek`，与设置中的学期总周数联动）、备注。
- 节次输入支持多分隔符解析：
  - 英文逗号 `,`
  - 中文逗号 `，`
  - 顿号 `、`
  - 分号 `;` / `；`
  - 空格
- 节次数据会自动：
  - 去重
  - 排序
  - 过滤到合法范围

### 5. 设置页（SettingsPage）

- 主题设置（跟随系统 / 浅色 / 深色）。
- **浅色模式下**设置内容区采用**外围略深、内层更浅**的双层卡片视觉层次；深色模式保持接近系统单卡片效果。
- 关闭行为设置（最小化到托盘 / 直接关闭）。
- 关闭提示开关。
- **课表方案**：多方案列表、新建/重命名/删除、当前方案切换（`schemes.json` + `schemes/{id}/` 目录）。
- 课程表显示范围设置（5 天 / 7 天）。
- **学期**：学期起始周日期（周一锚点）、学期总周数（与课表周次、导入解析等一致）。
- 每日节数设置（1~20）。
- 每节时间设置：
  - 当前使用“开始按钮 ~ 结束按钮”模式
  - 点击按钮弹出 TimePicker 进行时间选择
  - 选择结果回填到按钮文本
  - 使用 `HH:mm` 统一格式展示
- 节次时间数据使用结构化类型（开始/结束时间），并兼容旧字符串配置读取。
- **教务系统导入**：打开独立窗口（`ImportWebViewWindow.cs`，内嵌 WebView2）登录教务，从页面脚本提取课表并导入为**新方案**或合并进当前方案。
- 方案切换、教务导入等系统通知使用**静默**样式（无提示音、尽量仅进通知中心）。

### 6. 主导航（MainPage）

- 左侧 `NavigationView`：首页、课程表、课程列表、设置。
- 侧栏 **Compact / Expanded** 切换阈值可调（当前约 **520px / 768px**），使较窄窗口仍尽量保持侧栏展开体验。
- 侧栏展开/收起时，内容区使用系统**重排过渡**，减轻布局瞬间跳动感。

### 7. 数据与配置

- 课程数据持久化到各方案目录下的 `courses.json`。
- 课表相关配置持久化到各方案目录下的 `config.json`。
- 按周调课覆盖持久化到各方案目录下的 `week-overrides.json`。
- 方案列表与全局主题/关闭偏好等写入应用根目录 `schemes.json`（及历史文件迁移逻辑）。
- 保存机制包含去抖与并发写保护，降低高频操作下的数据覆盖风险。

### 8. 待办（ToDoPage）与置顶小窗（TodoPinWindow）

- 待办列表：卡片式展示，支持标题、内容、截止日期、优先级、标签与完成状态。
- 待办 Pin：每个待办卡片可点击 Pin 打开**置顶小窗**（Always-on-top），用于快速查看与勾选完成。
- 置顶小窗风格：使用 WinUI 3 窗口能力与 Mica 背景，主题跟随主窗口。
- Tab 管理：
  - 同一待办重复 Pin：会**聚焦**到已打开的 Tab（不会重复创建）。
  - Tab 支持跨窗口拖拽：可**拆分**为新窗口，也可拖到另一个窗口 TabStrip **合并**。
  - 当一个窗口最后一个 Tab 被合并拖走后，源窗口会自动关闭，避免空白窗口残留。
- 完成状态双向同步：
  - 主窗口勾选完成会同步更新对应置顶小窗。
  - 置顶小窗勾选完成会同步更新主窗口对应卡片。
  - 同步采用**增量更新**（仅更新对应待办的 Completed），避免全量刷新导致卡片选中/hover 抖动与勾选动画重放。
  - 主题切换后仍保持同步，不依赖旧页面实例回调。

### 9. 冲突检测

- 新增/编辑课程时执行时间冲突校验。
- 课表拖放按周/全局调整时，结合**当前周**的有效节次（含 `week-overrides`）做冲突判断。
- 冲突规则综合考虑：
  - 星期
  - 节次交集
  - 周类型（全周/单双周）
  - 学期内周次范围
- 发现冲突时弹窗提示并阻止保存或拖放落点。

### 10. 系统托盘与窗口

- 主窗口限制**最小宽度**（约 450px），避免极端窄窗下布局不可用。
- 标题栏提供**置顶开关**（Pin）：可在“窗口置顶 / 取消置顶”之间切换。
- 支持窗口关闭后最小化到系统托盘。
- 支持从托盘恢复窗口。
- 托盘右键菜单支持：**显示大窗口 / 显示小窗口 / 退出**。
- 托盘菜单触发的大/小窗口显示会联动窗口模式切换动画与标题栏高度切换。
- 托盘行为可在设置页实时切换。
- `ContentDialog` 经统一守卫 `ContentDialogGuard` 串行弹出，降低多页面异步流程下并发弹窗异常风险。

### 11. 主窗口近期架构调整

- `MainWindow.xaml.cs` 已将窗口模式切换动画逻辑下沉到 `Helpers/Ui/WindowModeController`。
- 托盘图标、右键菜单及隐藏/恢复状态流程已集中到 `Helpers/Platform/TrayMenuController`。
- 通过职责下沉，主窗口代码主要保留事件编排与页面级业务决策，维护成本更低。
