# CourseList 项目功能说明

## 项目定位

CourseList 是一个基于 WinUI 3 的课程管理与课表展示应用，核心目标是帮助用户快速录入课程、可视化查看课程安排，并提供首页今日课程概览与个性化设置能力。

## 代码与目录结构（概要）

- **`Models/`**：领域模型（`Course`、`WeekScheduleOverride`），详见 `Models/README.md`。
- **`Views/`**：各 WinUI 页面与教务导入窗口，详见 `Views/README.md`。
- **`Helpers/`**：业务与基础设施逻辑，按职责分子目录，**命名空间仍为 `CourseList.Helpers`**，详见 `Helpers/README.md`。
  - `Helpers/Core/`：数据根路径、多方案、`config.json` / `courses.json`
  - `Helpers/Schedule/`：按周有效课表、周覆盖文件、学期周次、冲突检测
  - `Helpers/Import/`：教务 WebView 会话与课表 JSON 解析
  - `Helpers/Ui/`：主题、对话框守卫、系统通知
  - `Helpers/Platform/`：系统托盘

## 已实现功能

## 1. 首页（HomePage）

- 显示欢迎信息、当天星期、今日课程概览。
- 显示“今日课程”卡片列表（独立列布局）。
- 卡片采用左侧色条 + 右侧信息样式，直观区分课程颜色。
- 当天课程会按节次自动排序。
- 对于同一课程在同一天的**不连续节次**，会拆分为多条课程安排分别显示。
- 时间展示支持紧凑格式，如 `08:00 ~ 09:40`。
- 切换课表方案后，首页数据随当前方案刷新。

## 2. 课程列表（CourseListPage）

- 课程增删改查（新增、编辑、删除）。
- 卡片式课程展示（固定卡片高度，卡片**内部**可滚动查看长文本），支持显示：
  - 课程名
  - 教师
  - 教室
  - 上课时间
  - 备注
- 列表区域外包 **`ScrollViewer`**：卡片较多超出窗口高度时出现**纵向滚动条**，避免内容被裁切且无滚动。
- 支持搜索（名称/教师/教室/备注/时间文本）。
- 支持筛选：
  - 星期筛选
  - 周类型筛选（全周、单周、双周）
  - 节次筛选
- 卡片选中高亮，便于编辑/删除操作。

## 3. 课程表页（SchedulePage）

- 网格式课程表展示（支持周一到周五 / 周一到周日两种模式，由设置决定）。
- 根据“每日节数”动态生成行数。
- 首列显示“节次 + 对应时间”。
- 浅色/深色下表头与背景使用页面内主题色，与系统主题协调。
- 课程渲染支持连续节次合并：
  - 连续节次视觉合并为一个课程块
  - 中间缝隙不再断开
  - 课程名称/教师/教室只显示一次
- 课程选中与交互：
  - 点击课程可选中并高亮
  - 双击课程可直接进入编辑
  - 点击空白区域取消选中
- 底部周次切换（上一周/下一周/回到本周），并显示是否“非本周”提示。
- 悬浮操作栏支持展开/收起动画（新增、编辑、删除）。
- **拖放排课**：将课程块拖到其他节次/星期；松手后可选「仅当前课表页显示的第 N 周」或「全局」生效，并进行时间冲突检测；拖拽过程含预览与平滑过渡。
- **按周调课**：仅本周的排课保存在当前方案目录的 `week-overrides.json`，不修改全局课程节次；全局调课则写入 `courses.json` 并移除该周对应覆盖。

## 4. 课程录入与编辑（CourseFormPage）

- 支持录入课程基础字段：课程名、教师、教室、星期、节次、颜色、周类型、备注。
- 节次输入支持多分隔符解析：
  - 英文逗号 `,`
  - 中文逗号 `，`
  - 顿号 `、`
  - 分号 `;` / `；`
  - 空格
- 节次数据会自动：
  - 去重
  - 排序
  - 过滤到合法范围

## 5. 设置页（SettingsPage）

- 主题设置（跟随系统 / 浅色 / 深色）。
- **浅色模式下**设置内容区采用**外围略深、内层更浅**的双层卡片视觉层次；深色模式保持接近系统单卡片效果。
- 关闭行为设置（最小化到托盘 / 直接关闭）。
- 关闭提示开关。
- **课表方案**：多方案列表、新建/重命名/删除、当前方案切换（`schemes.json` + `schemes/{id}/` 目录）。
- 课程表显示范围设置（5 天 / 7 天）。
- **学期**：学期起始周日期（周一锚点）、学期总周数（与课表周次、导入解析等一致）。
- 每日节数设置（1~20）。
- 每节时间设置：
  - 当前使用“开始按钮 ~ 结束按钮”模式
  - 点击按钮弹出 TimePicker 进行时间选择
  - 选择结果回填到按钮文本
  - 使用 `HH:mm` 统一格式展示
- 节次时间数据使用结构化类型（开始/结束时间），并兼容旧字符串配置读取。
- **教务系统导入**：打开独立 WebView 窗口登录教务，从页面脚本提取课表并导入为**新方案**或合并进当前方案（具体交互见 `ImportWebViewWindow`）。
- 方案切换、教务导入等系统通知使用**静默**样式（无提示音、尽量仅进通知中心）。

## 6. 主导航（MainPage）

- 左侧 `NavigationView`：首页、课程表、课程列表、设置。
- 侧栏展开/收起时，内容区使用系统**重排过渡**，减轻布局瞬间跳动感。

## 7. 数据与配置

- 课程数据持久化到各方案目录下的 `courses.json`。
- 课表相关配置持久化到各方案目录下的 `config.json`。
- 按周调课覆盖持久化到各方案目录下的 `week-overrides.json`。
- 方案列表与全局主题/关闭偏好等写入应用根目录 `schemes.json`（及历史文件迁移逻辑）。
- 保存机制包含去抖与并发写保护，降低高频操作下的数据覆盖风险。

## 8. 冲突检测

- 新增/编辑课程时执行时间冲突校验。
- 课表拖放按周/全局调整时，结合**当前周**的有效节次（含 `week-overrides`）做冲突判断。
- 冲突规则综合考虑：
  - 星期
  - 节次交集
  - 周类型（全周/单双周）
  - 学期内周次范围
- 发现冲突时弹窗提示并阻止保存或拖放落点。

## 9. 系统托盘与窗口

- 支持窗口关闭后最小化到系统托盘。
- 支持从托盘恢复窗口、托盘菜单进入首页等。
- 托盘行为可在设置页实时切换。
- `ContentDialog` 经统一守卫串行弹出，降低多对话框并发导致的异常风险。

## 未来可能功能更新

## 1. 课表体验增强

- 课程块大小/密度切换（紧凑、标准、大字模式）。
- 显示“当前节次 / 下一节课”实时状态高亮。
- 窄窗口下的横向滚动或列折叠策略优化。

## 2. 通知与日程联动

- 按场景配置系统通知（静音/有声、是否横幅）。
- 导出 iCal / 与系统日历联动。

## 3. 数据管理能力

- 课程数据导入/导出（JSON、CSV、Excel）与批量编辑。
- 课表模板（学期模板、班级模板）一键套用。
- 课程历史版本与误操作回滚。

## 4. 多周与学期维度

- 考试周、假期周等特殊课表覆盖或隐藏规则。
- 校历导入（节假日自动标注）。

## 5. 课程组织方式升级

- 同一课程多天排课的“主课程 + 多时段”建模（可联动编辑）。
- 课程标签系统（必修/选修/实验/线上）。
- 自定义分组视图（按教师、教室、课程类型）。

## 6. UI 与可访问性

- 更丰富的主题配色（自定义课程色板）。
- 键盘导航优化与屏幕阅读器适配。
- 高 DPI、触控屏与小屏窗口的响应式布局优化。

## 7. 可靠性与质量

- 增加自动化测试（冲突检测、时间解析、布局关键逻辑）。
- 增加日志与错误追踪能力，便于定位线上问题。
- 增加启动健康检查（配置迁移、数据完整性检查）。
- 将开发用硬编码数据路径（如 `PathHelper.BaseFolder`）改为按用户/安装目录可配置或包内相对路径。
